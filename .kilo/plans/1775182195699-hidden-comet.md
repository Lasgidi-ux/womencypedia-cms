# Fix Strapi API 404 Errors Plan

## Problem Analysis
**PERSISTENT ROOT CAUSE**: Core Strapi plugins not functioning despite server startup:
- ✅ **Server starts successfully** - No more plugin loading errors
- ✅ **Admin panel loads** - Basic UI accessible at `/admin`
- ✅ **Content types registered** - All 37 content types loaded in bootstrap
- ✅ **Explicit routes created** - Manual API route definitions in place
- ❌ **Content-manager plugin broken** - `/content-manager/*` endpoints return 404
- ❌ **Authentication failing** - `/admin/users/me/*` endpoints return 401
- ❌ **Admin CRUD operations fail** - Cannot create/edit/delete content

**Current Status:**
- Cache cleared successfully, server starts without errors
- Admin panel loads but content management doesn't work
- Public API routes may work, but admin functionality is broken
- Core plugins (content-manager, auth) appear non-functional

## Root Cause Investigation
1. **Content Type Registration**: Check if content types are properly registered in the database
2. **Route Loading**: Verify routes are auto-loading in Strapi v5
3. **Server State**: Ensure production server has latest code and database migrations
4. **Strapi v5 Compatibility**: Confirm API patterns match v5 requirements

## Solution Options

### Option 1: Enable Content-Manager Plugin Explicitly
- **Add content-manager to plugins config** (despite being "core"):
  ```typescript
  pluginConfig['content-manager'] = { enabled: true };
  ```
- **Verify plugin exists** in Strapi v5 core
- **Check if additional configuration** is needed

### Option 2: Check Strapi v5 Plugin Architecture
- **Research Strapi v5 plugin system** - may have changed from v4
- **Verify core plugins** are enabled by default or need explicit config
- **Check official documentation** for v5 plugin configuration

### Option 3: Database Reset and Content Type Re-sync
- **Clear database** to remove any corrupted plugin registrations
- **Re-run bootstrap** to re-register content types and permissions
- **Force content type sync** with database schema

### Option 4: Fresh Strapi v5 Installation (Last Resort)
- **Create new Strapi v5 project** with proper plugin setup
- **Migrate content types** and data from current project
- **Ensure all plugins** work correctly from start

## Implementation Steps

### Phase 1: Enable Content-Manager Plugin
1. **Add content-manager plugin to config**:
   ```typescript
   // In config/plugins.ts
   pluginConfig['content-manager'] = {
     enabled: true
   };
   ```
   - Explicitly enable despite being considered "core"

2. **Check if other core plugins need enabling**:
   - Add `content-type-builder`, `upload` if needed
   - Test minimal configuration first

3. **Build and deploy**:
   - Commit changes and push to trigger deployment
   - Monitor for content-manager endpoints working

### Phase 2: Alternative - Strapi v5 Plugin Research
If explicit enabling doesn't work:

1. **Research Strapi v5 plugin architecture**:
   - Check if plugins are enabled differently in v5
   - Verify core plugins are auto-enabled
   - Look for v5-specific configuration requirements

2. **Test with different plugin configurations**:
   - Try enabling one plugin at a time
   - Check Render.com logs for specific error messages

### Phase 3: Database and Content Type Reset
If plugins still don't work:

1. **Reset database** (backup first):
   - Clear corrupted plugin registrations
   - Re-run bootstrap to re-sync content types

2. **Force content type re-registration**:
   - Ensure all content types are properly loaded
   - Verify permissions are correctly set

### Phase 4: Fresh Installation (Final Option)
1. **Create new Strapi v5 project** with proper setup
2. **Import configurations** and content types
3. **Test all functionality** in clean environment
4. **Migrate data** if needed
